# AI 语音绘图工具设计文档

## 1. 目标与约束

题目要求开发一款纯语音控制的绘图工具。核心目标是：用户不通过鼠标或键盘进行绘图创作，只用语音完成唤醒、绘制、修改和停止。

本项目将“纯语音”定义为：

- 首次授权麦克风可能需要一次浏览器点击。
- 绘图创作流程中，用户不需要鼠标拖拽、不需要键盘输入。
- 画布内容的新增、撤销、重做、清空均由语音指令触发。

## 2. 计划支持的能力

### 2.1 唤醒与对话

- 唤醒词：“小花小花”。
- 误识别容错：“小花”“笑话”“小化”“消化”。
- 唤醒后语音回复：“我在，请问有什么可以帮您的？”
- 支持“一口气说完”：例如“小花小花，画一个太阳”。

### 2.2 绘图指令

- 基础形状：圆形、方形、长方形、三角形、线条。
- 组合物体：房子、树、云、太阳、船、花、爱心、星星。
- 场景背景：天空、背景色。
- 颜色：红、蓝、绿、黄、橙、紫、粉、黑、白、灰、棕、金、银、青等。
- 位置：左边、右边、上方、下方、中间、中心。
- 大小：大、小、巨大、很小。

### 2.3 编辑指令

- 撤销：撤销、撤回、上一步。
- 重做：重做、恢复、下一步。
- 清空：清空、清除、清屏、全部清除、擦除全部。
- 停止：停止、停下来、不用画了、别画了、取消、算了。

### 2.4 复杂指令拆解

计划支持一句话包含多个绘图目标：

- “画蓝天白云和太阳”
- “画一座房子和一棵树”
- “在左边画树，在右边画房子”

实现策略是先用本地规则做低延迟解析；本地无法解析时再请求后端大模型解析 DSL。

## 3. 当前实现

### 3.1 语音识别链路

前端使用浏览器 `SpeechRecognition` 监听中文语音，使用 `SpeechSynthesis` 做语音回复。为了避免 TTS 被麦克风再次识别，回复时会暂停语音输入，回复结束后再恢复监听。

状态流：

```text
sleeping -> waking -> listening -> processing -> speaking -> listening
```

### 3.2 绘图执行链路

识别到最终文本后，前端按以下顺序处理：

1. 判断是否为停止类指令。
2. 使用本地规则解析常见绘图和编辑命令。
3. 本地无法解析时，调用后端大模型解析为绘图 DSL。
4. 如果仍无法解析，则使用图片生成服务作为兜底。
5. 将 DSL 操作写入 Zustand 画布状态，由 Canvas 2D 渲染。

### 3.3 已实现能力

- “小花小花”唤醒与语音应答。
- 本地解析圆形、矩形、三角形、天空、房子、树、云、太阳、船、花、爱心、星星。
- 支持颜色、简单位置、简单大小。
- 支持多元素指令拆解。
- 支持撤销、重做、清空和停止。
- 支持指令历史与画布对象列表展示。
- 支持后端大模型解析和图片生成兜底接口。

## 4. 未完成与原因

- 复杂空间关系仍有限：例如“把树移动到房子后面”需要更完整的对象引用和层级关系模型。
- 连续对话改图仍有限：例如“把刚才那个变大一点”需要更强的上下文指代解析。
- 云端 ASR 未作为默认路径：为了降低部署复杂度和响应延迟，当前默认使用浏览器语音识别。
- 公网部署待完成：本地功能可运行，但正式提交前仍需要部署前端和后端，并在 README 补充在线体验与 Demo 视频链接。

## 5. 测试方式

### 5.1 本地构建

```bash
cd frontend
npm run build
```

### 5.2 手动语音测试

1. 打开前端页面并允许麦克风。
2. 说“小花小花”。
3. 听到回复后说“画蓝天白云和太阳”。
4. 确认画布出现背景、云和太阳，指令历史出现记录。
5. 继续说“撤销”“重做”“清空画布”验证编辑能力。
